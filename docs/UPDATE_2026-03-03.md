# Update Notes (2026-03-03)

## What was added

### 1) Reliability improvements for fetching
- URL normalization for WeChat links
- Multi-profile request retries
- Block-page detection
- Optional remote fallback (no sudo)
- Optional browser fallback (Playwright)

### 2) PDF preview-first workflow
- Added in-browser PDF preview endpoint: `/api/view/<filename>`
- UI now supports direct preview after generation

### 3) Save path automation
- Save mode options:
  - default
  - by author
  - custom fixed path
- Remember preferences in config
- Author-to-path mapping support
- New APIs:
  - `GET/POST /api/save-prefs`
  - `GET/POST/DELETE /api/author-paths`

### 4) Browser print mode
- Optional `use_browser_print` flow to generate PDF using browser print (`page.pdf`)
- Intended for pages that render better via browser pipeline

## Important behavior
- `start_server.py` now starts **PRO** server by default.
- Demo server is no longer the default path.

## Compatibility notes
- Browser fallback/print needs Playwright runtime and required system libraries.
- If system dependencies are missing, use remote fallback mode.
